## 📍 你当前的状态

你刚刚运行了 `npm run upload:standalone`，但看到了这个错误：

```
❌ 创建 bucket 失败: new row violates row-level security policy
```

**这是完全正常的！** ✅

## 🤔 为什么会这样？

Supabase 出于安全考虑，不允许通过 API 自动创建 Storage bucket。你需要手动在 Supabase Dashboard 中创建。

这只需要做**一次**，之后就可以随时上传更新了。

## 👉 接下来的步骤（5分钟完成）

### Step 1: 打开 Supabase Dashboard

1. 打开浏览器
2. 访问：https://app.supabase.com
3. 登录你的账号
4. 选择项目：`rimhmaeecdcrhuqbisjv`

### Step 2: 创建 Bucket

1. 点击左侧菜单的 **"Storage"** 
2. 点击右上角 **"New bucket"** 按钮
3. 填写：
   - Name: `public-pages` ⚠️（必须完全一致）
   - ✅ 勾选 **"Public bucket"**
4. 点击 **"Create bucket"**

### Step 3: 重新运行上传命令

回到终端，再次运行：

```bash
npm run upload:standalone
```

### Step 4: 成功！

这次你会看到：

```
✅ 部署成功！

📍 稍后阅读主页 URL:
   https://rimhmaeecdcrhuqbisjv.supabase.co/storage/v1/object/public/public-pages/read-later.html
```

## 🎉 完成后

1. **复制**上面显示的 URL

## ❓ 还有问题？

### Q: 我必须手动创建 bucket 吗？
**A**: 是的，这是 Supabase 的安全限制，但只需要做一次。

### Q: bucket 名称可以改吗？
**A**: 不建议。如果一定要改，需要同时修改上传脚本中的配置。

### Q: 创建后可以改成 private 吗？
**A**: 可以，但页面将无法公开访问。如果需要私有访问，需要额外配置认证。

### Q: 以后更新需要重新创建 bucket 吗？
**A**: 不需要！bucket 创建一次后，以后只需要运行 `npm run upload:standalone` 即可更新。
